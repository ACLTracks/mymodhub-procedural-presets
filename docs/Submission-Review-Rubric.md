Submission Review Rubric

This rubric is used to evaluate community packs for acceptance, featuring, or rejection.

It prioritizes stability, clarity, and ecosystem health over hype.

1. Technical Validity

Pass / Fail

Schema compliant

Loads without errors

Applies deterministically

Does not break other packs

Failure here is an automatic rejection.

2. Safety and Respect for User Choice

Score: 0 to 5

0
Overrides head parts or ignores user setup

3
Mostly safe but pushes boundaries

5
Fully additive, respectful, and modular

3. Intent Clarity

Score: 0 to 5

0
Unclear what the pack is trying to do

3
Intent exists but is muddled

5
Clear, focused, and well described

4. Design Quality

Score: 0 to 5

0
Extreme, unstable, or poorly balanced

3
Acceptable but rough around the edges

5
Balanced, grounded, and stack-friendly

5. Metadata and Documentation

Score: 0 to 5

0
Missing or misleading metadata

3
Adequate but minimal

5
Clear descriptions, good tags, helpful notes

6. Compatibility Awareness

Score: 0 to 5

0
Assumes niche setups or breaks common ones

3
Works in most cases

5
Designed to work broadly and gracefully

7. Ecosystem Value

Score: 0 to 5

0
One-off novelty with no remix value

3
Useful but limited

5
Highly reusable building block

Review Outcomes

Rejected

Fails technical validity

Breaks user control rules

Misrepresents behavior

Accepted

Meets minimum technical and safety standards

Featured

High scores across most categories

Clean design

Clear intent

Strong remix potential

Philosophy of Review

This is not a popularity contest.

Packs are judged on:

stability

clarity

respect for users

contribution to a scalable system

The goal is a healthy ecosystem, not endless noise.
